Speed & Scalability

Processing hundreds of resumes manually is infeasible. Automation scales to thousands of applications without degradation in quality.

Consistency

Human reviewers exhibit fatigue and cognitive bias. A rule-based system applies identical criteria to every resume — no implicit preferences.

Candidate Feedback

Commercial ATS provides zero feedback to applicants. This system generates a personalised gap report and learning roadmap — supporting both recruiters and job seekers.

02 Objectives

8 Core Objectives

Defined in Chapter 1.3 of the mid-semester report — each objective maps directly to a system module.

O1 — Multi-Format Parsing

Parse resumes from PDF, DOCX, TXT and scanned images via OCR.

O2 — Information Extraction

Extract name, email, phone, skills, work experience, education, certifications.

O3 — JD Analysis

Parse job descriptions to identify required skills, preferred skills, and experience requirements.

O4 — Weighted Matching

Compute a multi-dimensional match score to help strengthen decision-making, grounded in real-life recruitment scenarios and logically weighted criteria.

O5 — Gap Analysis

Identify skill and experience gaps; generate a prioritised learning path with specific resources.

O6 — Web Interface

Streamlit web app with 3 tabs: Resume Analysis · Job Matching · Gap Analysis; plus contextual email dispatch for:
  • Approval — interview invitation with scheduling details
  • Rejection — detailed reasoning and personalised improvement points
  • Review — resume forwarded to hiring manager for final decision

O7 — Unit Testing

Validate every module through automated pytest tests across multiple test files.

O8 — Quantitative Evaluation

Measure accuracy using Precision, Recall, F1-Score per module and Spearman's ρ for matching.
03 Review of Literature

Prior Work: Academic Research

Chapter 2 of the mid-semester report surveys 6 key academic papers on NLP-based resume analysis.

Reference Approach Key Contribution Gap / Limitation
Faliagka et al. (2012) ML + Semantic Web Ontology-based candidate ranking Domain-specific; no multi-format support
Maheshwary & Misra (2018) TF-IDF + Cosine Similarity Vector space model for resume–JD matching Ignores phrase context; no gap analysis
Luo et al. (2019) BERT fine-tuned on HR data Semantic similarity for job–candidate matching Large labelled corpus required; computationally expensive
Roy et al. (2020) spaCy NER + Word2Vec Skill and entity extraction with word embeddings Custom NER training required; no OCR support
Zhang & Wang (2021) Graph Neural Networks Skill taxonomy via graph-based matching Highly complex; limited to structured data
Gugnani & Misra (2020) LSTM + CRF Resume section segmentation & NER No end-to-end usable system delivered
Synthesis — Research Gap
Academic work focuses on individual components (NER, matching, embeddings) but delivers no complete, runnable, multi-modal, open-source system. Deep-learning approaches (BERT, GNN) achieve higher semantic accuracy but require large labelled corpora and significant compute — impractical for this dissertation scope. The present work occupies the "pragmatic middle ground": rule-based NLP with optional spaCy, covering the full pipeline end-to-end.

Matching Score Formula
Overall Score = 0.60 × Skill Score
                + 0.30 × Experience Score
                + 0.10 × Education Score
                × Confidence Factor

Skill Score = 0.70 × (matched_req / total_req)
               + 0.30 × (matched_pref / total_pref)

Exp. Score = min(1.0, candidate_yrs / required_yrs)
Edu. Score = 1.0 if degree matches | 0.5 if any degree | 0.0 none

Similarity Metrics (SimilarityCalculator)
  • Jaccard — set intersection over union
  • Cosine — TF-IDF vector angle
  • SequenceMatcher — longest common subsequence ratio
  • Overlap (Szymkiewicz–Simpson) — smaller set coverage
  • fuzzy_match() — near-synonym catching (e.g. "ML" ↔ "Machine Learning") at configurable threshold
07 Evaluation Metrics

Quantitative Evaluation Framework

Standard NLP metrics measure extraction quality; matching metrics measure shortlisting accuracy. All implemented in src/evaluation/.

Extraction Metric Definitions
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1-Score = 2 × P × R / (P + R)

TP = correctly extracted entity
FP = extracted but incorrect / not in ground truth
FN = in ground truth but not extracted
Ground Truth
Hand-annotated labels for 2 sample resumes (Jane Smith, Michael Chen) covering all 7 extraction modules. Defined in src/evaluation/ground_truth.py. Recruiter reference decisions used for matching evaluation.
